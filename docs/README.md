# docs

## Purpose

Operational documentation for architecture, standards, and workflows.

## What belongs here

- Architecture decisions and structure docs
- Quality and naming standards
- Repeatable contributor workflows

## What does not belong here

- Bot runtime files
- Generic snippets
- Shared executable helpers

## Naming rules

- Use concise kebab-case filenames.
- Prefer one focused topic per file.

## Agent workflow rules

- Keep docs short and actionable.
- Update docs when structure or process changes.
