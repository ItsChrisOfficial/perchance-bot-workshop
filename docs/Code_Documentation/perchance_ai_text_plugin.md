# Perchance AI Text Plugin

This document summarises the **AI text plugin** for Perchance.  The plugin enables server‑side text generation using large language models.  Because the model runs on GPU servers rather than in your browser, it may display ads for non‑logged‑in users to cover hosting costs.

## Importing the plugin

To use the AI text plugin, import it in your lists editor:

```text
ai = {import:ai-text-plugin}
```

You can then call the plugin via square brackets.  For example, generate a poem about a character in a place during a season:

```text
character
  {mech|demon|cyberpunk} {warrior|minion|samurai}

place
  a retropunk distopia
  a small village
  a mountainous region
  an underwater cavern

season
  winter
  summer

poemPrompt
  instruction = Write a haiku about a [character] in [place] during [season].

output
  [ai(poemPrompt)]
```

Calling `[ai(prompt)]` returns text generated by the AI model based on your prompt or instruction.  You can embed the result anywhere in your generator by writing `[output]`.

## Basic usage

* **Instruction** – Provide the task as an instruction (e.g., `instruction = Write a story about …`).
* **StartWith** – Specify a starting string that the AI must begin with (e.g., `startWith = It was the night before Christmas in`).
* **Direct call** – You can pass plain text directly into the plugin: `[ai("Explain quantum field theory to a toddler.")]`.
* **See the instruction on hover** – A small icon appears at the end of the AI response; hover over it to see the instruction that was used.

## Advanced options

The plugin supports many options to control generation:

- **instruction** – A string describing what to write.
- **startWith** – Required starting text.  The AI will continue from this prefix.
- **stopSequences** – List of words or phrases that halt generation if encountered.
- **hideStartWith** – If `true`, the `startWith` text is hidden from the final output.
- **outputTo** – Direct the output to an element by ID (e.g., `outputTo = [myElement]`).  Otherwise, the plugin outputs inline.
- **onChunk(data)** – JavaScript code executed after each chunk (partial generation).  The `data` object contains `textChunk`, `fullTextSoFar`, and `isFromStartWith`.
- **onStart(data)** – Code run at the start of generation.  Access inputs via `data.inputs.instruction`, etc.
- **onFinish(data)** – Code run when generation completes.  Access the final `data.text` (including the startWith text) and `data.generatedText` (excluding it).
- **render(data)** – Custom render function executed after each chunk.  Return a string to transform the displayed output (e.g., convert `*smiles smugly*` into bold or italic HTML).
- **endButtons** – Set to `none` to remove the edit/continue buttons that appear after the response.

All option values may be functions, allowing dynamic instructions.  For example, the `instruction` can be a function that returns a string based on user input.

## Styling output

You can provide a `style` option in your prompt data to apply CSS styles to the output.  For instance:

```text
marioAffirmationsPrompt
  instruction = Be Mario, and give me 3 positive affirmations with Mario's accent.
  style = text-align:left; color:blue; font-weight:bold; border:2px solid red; display:block; max-width:600px; margin:0 auto; padding:0.5rem;
```

The AI’s response will then be rendered with these styles applied.

## Examples and advanced features

The plugin documentation includes numerous example generators demonstrating advanced features such as:

- Combining image and text generation (using both `ai` and `text-to-image-plugin`).
- Prompt testers to quickly iterate on AI instructions.
- Two-character chats where two characters converse automatically.
- Short stories with embedded images and custom rendering.
- Text‑to‑speech integration via the AI text plugin’s `onChunk` callback.
- Multi‑choice text adventures and hierarchical world explorers.
- Handling user input as part of the instruction.

You can also specify negative prompts, guidance scale values, and custom rendering functions to refine output.  See the plugin examples on Perchance for demonstrations of these advanced features.

## Notes

- Text prompts and responses are **not stored on the server** unless explicitly saved to a gallery.  Each generation is transient.
- The plugin may produce NSFW content if the prompt requests it; use responsible prompting and add negative prompts to filter out unwanted themes.
- Non‑logged‑in users will see ads below your generator to cover server costs.  Logged‑in users will not see ads.
- Each user can have only a few concurrent server requests.  If multiple completions are pending, they will queue up.

For more examples and advanced usage patterns, explore the [AI text plugin documentation on Perchance](https://perchance.org/ai-text-plugin) and community resources.
