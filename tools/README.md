# tools

## Purpose

Repository tooling notes and support files.

## What belongs here

- Tool configuration references
- Operational helper docs for development tooling

## What does not belong here

- Bot project files
- Shared runtime helper code (use `shared/utilities/`)

## Naming rules

- Use explicit kebab-case names.
- Keep one concern per file.

## Agent workflow rules

- Keep tool docs concise and actionable.
- Do not add heavy dependencies without clear need.
