# Project Rules

1. Keep root clean; only high-level governance/index files belong there.
2. Use lifecycle separation for bots: `templates`, `in-progress`, `completed`.
3. Prefer shared reusable assets over copy-paste.
4. Keep documentation concise and operational.
5. Use predictable naming (kebab-case, no ambiguous folder names).
6. Preserve structure unless refactoring is intentional and documented.
7. Update indexes (`REPO_MAP.md`, `BOT_CATALOG.md`, `SNIPPETS_INDEX.md`) when adding major assets.
